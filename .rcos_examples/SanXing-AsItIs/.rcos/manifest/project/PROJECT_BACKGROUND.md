# PROJECT_BACKGROUND

## 项目名称
三省 Check-in（SanXing Digital Check-in）

## 项目定位

一个将传统“三省书”修行流程数字化的低摩擦自我反思系统。

该系统以佛教修行体系（八正道及相关行为维度）为基础，通过每日三次结构化 check-in，实现：

- 行为自我观察
- 语义化评分
- 长期记录与复盘

---

## 项目核心目标

- 将纸质三省 workbook 转换为数字化交互系统
- 降低记录成本（按钮替代书写）
- 保持原有修行体系（不改变语义）
- 支持多设备访问（手机 / PC）
- 支持数据导出回原始 PDF 结构

---

## 项目本质（RCOS视角）

该项目不是简单的 CRUD 系统，而是：

👉 一个“行为语义建模 + 反思执行系统”

其核心挑战在于：

- 将《三省书》扫描件中的修行标准
- 转换为可执行的评分模型（scoring model）

当前项目已经有一份初始评分语义种子：

- `.rcos/manifest/project/scoring_model_seed.json`

以及其主要参考材料：

- `.rcos/manifest/project/三省书_monthly.pdf`

当前确认状态：

- seed JSON 已形成一个可工作的语义基线
- 其结构与项目目标一致：三次时间切片、预读内容、12 个评分维度、正负向统一高分语义
- 但由于参考 PDF 是扫描件，且本机当前没有 OCR / PDF 文本提取工具，尚未完成逐字级重新转录核对

因此当前应将该 seed 视为：

👉 已可用于继续建模的工作基线，而不是已完成逐项源文校勘的最终定稿

---

## RCOS 测试目的（非常关键）

本项目用于验证 RCOS 在以下场景下的能力：

1. 语义核心缺失情况下的渐进建模能力
2. 多层结构系统（理念 → 行为 → UI）的上下文一致性
3. AI 在受控 context 下的协作稳定性
4. 防止 scope 漂移（避免直接产品优化）

---

## 当前开发策略

优先级：

1️⃣ 语义建模（评分体系）
2️⃣ 数据结构与 API
3️⃣ UI 实现
4️⃣ 扩展功能（导出 / 分析）

---

## 明确非目标（当前阶段）

- 不追求 UI 精致
- 不引入复杂用户系统
- 不扩展修行体系（不新增维度）
- 不做 AI 智能分析（暂缓）

---

## 一句话总结

这是一个：

👉 以八正道行为语义为核心的低摩擦自我校准系统

当前 project-specific context 还包含一个初始评分语义文件：
- `.rcos/manifest/project/scoring_model_seed.json`
用于承载从《三省书》扫描件及项目上下文中整理出的行为维度、锚点与时间切片结构，作为后续代码实现和 UI 设计的语义起点。
